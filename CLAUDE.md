## Project Overview

**tauri-voice-plugin**: A Tauri plugin providing Speech-to-Text (STT) transcription and optional integration with `tauri-plugin-recorder` for recording workflows. The plugin is a hybrid Rust/TypeScript/Python system with both backend transcription and frontend API bindings.

## Key Architecture Concepts

### Plugin Layer (Rust)
- **Commands**: Handler functions invoked from JavaScript via Tauri. All commands are async.
  - **STT commands**: `bootstrap_stt`, `transcribe_file`, `stt_health`, `setup_record_transcribe_pipeline`
  - **Recorder bridge commands**: `initialize_recorder_runtime`, `start_recording`, `stop_recording`, etc. (conditionally compiled with `recorder-bridge` feature)
- **Feature gates**: `recorder-bridge` (default-enabled) controls whether the plugin can proxy recorder commands.
- **Plugin state**: Managed via Tauri's state system; accessible via `app.tauri_plugin_stt()` extension trait.

### Transcription Workflow
1. Bootstrap: Creates Python venv and installs locked dependencies (`requirements-stt.lock.txt`).
2. Health: Checks readiness of venv, sidecar (`uv`), Python binary.
3. Transcribe: Spawns Python sidecar script (`scripts/transcribe.py`) to run inference.
4. Model allowlist: Only `tiny`, `tiny.en`, `base`, `base.en`, `small`, `small.en`, `medium`, `medium.en`, `large`, `large-v1`, `large-v2`, `large-v3`, `large-v3-turbo`, `turbo`.

### Recording & Hotkeys (Desktop)
- **Recorder bridge** (`recorder-bridge` feature, enabled by default): The plugin proxies calls to `tauri-plugin-recorder` (sibling repo at `../tauri_recorder_plugin`).
- **Hotkey pipeline** (`setup_record_transcribe_pipeline`): Registers a single hotkey to toggle recording and auto-transcribe on completion.
- **Overlay coordination**: Displays 2-second transcript after transcription via event channel.

### TypeScript/JavaScript Frontend
- **guest-js/index.ts**: Exports command wrappers (`bootstrapStt()`, `transcribeFile()`, etc.) and event listener helpers.
- Built via rollup into ESM and CJS in `dist-js/`.
- Consumed as a local npm dependency in target apps.

## Common Development Tasks

### Build the Plugin

#### Rust backend:
```bash
cd tauri-plugin-stt
cargo build
```

#### TypeScript frontend:
```bash
cd tauri-plugin-stt
pnpm install
pnpm build
```

Both must be built for integration into host apps.

### Run Tests

#### Smoke contract tests (validate payload shapes):
```bash
cd tauri-plugin-stt
cargo test --test smoke_contract
```

#### Run the example Tauri app:
```bash
cd tauri-plugin-stt/examples/tauri-app
pnpm install
pnpm tauri dev
```

### Run a Single Test
```bash
cd tauri-plugin-stt
cargo test --test smoke_contract <test_name> -- --nocapture
```

### Lint & Check

```bash
cd tauri-plugin-stt
cargo clippy
cargo fmt --check
pnpm run build  # (runs rollup, reports TypeScript errors)
```

## Core Principles (from AGENTS.md)

### Prime Directive
Highest priority: readability. Do not extend functionality, integrate new interfaces, or update unrelated modules.

### Scope Control
- If the user asks for an interface, function, class, or file, generate code **only for that unit**.
- Do not touch consumers, pipelines, watchers, repositories, or API routes unless the user directly specifies them.
- Never infer "next steps" or perform proactive integration.
- If creating layers of abstractions or redirections, ask for confirmation first.

### Modification Rules
- Keep changes local to the specified path or file.
- No cross-module edits unless the user directly specifies them.
- No automated refactors.
- No updating unrelated parts of the system unless explicitly directed.

### Output Discipline
- Add clear comments that describe code concisely; max 10 words per comment.
- For newly generated code, include brief inline comments (separate line + blank line).
- Provide only the code asked for.
- Produce minimal diffs.
- Avoid adding abstractions, helpers, new dependencies, or reorganizing existing code.

### Documentation Standards
- **Docstrings**: Multi-line format with detailed description of purpose and behavior.
- **Parameters**: `:param name: description` for all parameters.
- **Returns**: `:return: description` describing return value and type.
- **Exceptions**: `:raises ExceptionType: condition` for documented exceptions.
- **Inline comments**: On separate lines before logical blocks, max 10 words, with blank line after.

## Plugin-Specific Patterns

### Command Contracts
- All commands must have request and response types in `models.rs`.
- Error responses use tagged enums (e.g., `BootstrapResponse::Ready { details }`, `BootstrapResponse::AlreadyReady { details }`).
- Commands are declared in `build.rs`; missing declarations will break the build.
- TypeScript wrappers in `guest-js/index.ts` must match the Rust contract.

### Feature Gates
- `recorder-bridge` (default-enabled): Allows recorder proxy commands. When disabled, those commands return `feature_disabled` error.
- Test with and without:
  ```bash
  cargo test --no-default-features
  ```

### Permissions
- Tauri requires explicit permission grants in `capabilities/default.json` (in host apps).
- This repo provides permission templates in the `permissions/` directory.

### Python Runtime
- Python 3.12 pinned.
- Virtual environment: `APP_DATA_DIR/python/.venv`.
- Transcription script: `scripts/transcribe.py` (spawned as sidecar).
- Lock file: `requirements/requirements-stt.lock.txt` (must be updated if dependencies change).
