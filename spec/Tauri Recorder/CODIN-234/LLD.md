# Low Level Design: Plugin Scaffold via Tauri CLI (CODIN-234)

## 1. Purpose

Define the minimum implementation for `CODIN-234` using Tauri CLI scaffolding, then only hand-write files that the scaffold does not provide.

This story is scaffold-only. No bootstrap or transcription runtime logic is implemented.

## 2. Locked Constraints

- Parent alignment: `CODIN-233`.
- Runtime lock: Python `3.14`.
- Platform lock: macOS Apple Silicon (`aarch64-apple-darwin`).
- Scope lock: STT-only, request/response shape, sidecar-only execution policy.
- Repository root: `/Users/karthik/merge_conflicts/voice_plugin`.

## 3. Scaffold Strategy

Primary approach:
- Generate plugin scaffold with Tauri CLI.

Allowed command forms:
- `cargo tauri plugin new tauri-plugin-stt`
- `npx @tauri-apps/cli plugin new tauri-plugin-stt`

Rule:
- Prefer generated scaffold defaults.
- Do not hand-create files that Tauri CLI already generates.

## 4. Hand-Written Changes Only

After CLI generation, limit manual edits to:

- Add `uv` sidecar packaging placeholders in generated config where needed.
- Add placeholder command names aligned to parent contract:
  - `bootstrap_stt`
  - `transcribe_file`
  - `stt_health`
- Add typed error code placeholders:
  - `bootstrap_failed`
  - `dependency_install_failed`
  - `model_download_failed`
  - `generation_failed`
  - `invalid_input`
  - `not_ready`
- Add `requirements/requirements-stt.lock.txt` as placeholder only.
- Add minimal README notes:
  - Python lock `3.14`
  - Canonical paths:
    - `APP_DATA_DIR/python/.venv`
    - `APP_DATA_DIR/python/.venv/bin/python`
    - `APP_DATA_DIR/models`
    - `APP_DATA_DIR/logs`

## 5. What Not To Hand-Write

- Do not manually author base plugin scaffolding files if generated by CLI.
- Do not create custom host app examples unless required by generated scaffold flow.
- Do not add runtime subprocess execution logic.
- Do not add bootstrap install logic.
- Do not add transcription logic.

## 6. Implementation Sequence

1. Run Tauri CLI plugin scaffold in current repo root.
2. Inspect generated output and keep defaults intact.
3. Apply only manual edits listed in Section 4.
4. Validate compile/scaffold integrity.

## 7. Validation Plan

- Generation check:
  - Plugin scaffold is produced by Tauri CLI.
- Minimal edit check:
  - Only Section 4 items are manually added.
- Contract check:
  - Placeholder command names are present.
- Policy check:
  - No command executes sidecar or Python in this story.
- Documentation check:
  - README records Python `3.14` and canonical app-data paths.

## 8. Exit Criteria for CODIN-234

- Scaffold is CLI-generated, not hand-built.
- Required placeholder command and error surfaces exist.
- `uv` sidecar packaging placeholder is present.
- Placeholder lock-file path exists.
- No functional bootstrap/transcribe behavior is added.
